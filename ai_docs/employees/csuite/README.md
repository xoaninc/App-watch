# C-Suite Executives

Executive leadership team for RenfeServer. Each role provides strategic guidance and decision-making authority in their domain.

## The Team

| Role | Focus Area | Key Decisions |
|------|------------|---------------|
| [CEO](ceo.md) | Vision & Strategy | Direction, funding, partnerships |
| [CTO](cto.md) | Technology | Architecture, stack, engineering |
| [CFO](cfo.md) | Finance | Pricing, metrics, fundraising |
| [CMO](cmo.md) | Marketing | Brand, growth, positioning |
| [CPO](cpo.md) | Product | Roadmap, features, UX |
| [CLO](clo.md) | Legal | Contracts, compliance, IP |
| [CAIO](caio.md) | AI Strategy | AI governance, ethics, tools |

## Decision Matrix

| Decision Type | Primary | Consulted | Informed |
|---------------|---------|-----------|----------|
| Product features | CPO | CTO, CMO | CEO |
| Technical architecture | CTO | CPO | CEO |
| Pricing changes | CFO | CPO, CMO | CEO |
| Marketing campaigns | CMO | CPO, CFO | CEO |
| Legal/compliance | CLO | CEO, CAIO | All |
| AI adoption | CAIO | CTO, CLO | CEO |
| Strategic direction | CEO | All | Board |

## How to Use

1. **Single Decision:** Ask the relevant executive directly
   ```
   "Ask the CTO: Should we use Redis or PostgreSQL for caching?"
   ```

2. **Cross-functional:** Convene relevant executives
   ```
   "Get CFO and CMO input on the freemium pricing strategy"
   ```

3. **Strategic Review:** Full C-Suite discussion
   ```
   "C-Suite: Review the Q2 roadmap priorities"
   ```

## Interaction Guidelines

- Each executive thinks from their functional perspective
- They may disagree - that's healthy tension
- CEO has final decision authority when needed
- Always consider trade-offs between functions
